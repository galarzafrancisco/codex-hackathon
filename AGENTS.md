# Agent Workflow

Agents must follow these guidelines before changing code in this repository.

## Backend Changes

- Read `docs/agent-guidelines/backend-layering.md` before making backend changes.
- Keep HTTP concerns in controllers.
- Keep DTO validation and OpenAPI annotations at the API boundary.
- Keep services transport-agnostic and use pure service input/result types.

## Frontend Changes

- Read `docs/agent-guidelines/frontend-api-and-style.md` before making frontend changes.
- Use the autogenerated API client instead of handwritten HTTP calls.
- Reuse `tokens.css`, shared primitives, and existing style tokens.
- Do not invent hardcoded styles per component.

## Local Validation

Before committing any change, agents must validate the app from the repository root:

```bash
npm run build
npm run start:dev
```

`npm run start:dev` should be used as a smoke test that the app starts successfully. Stop the dev process after confirming the app boots.

## Push and CI

- After pushing changes, inspect the CI build results.
- If any build or check fails, fix the failure locally.
- Re-run the local validation commands.
- Push the fix and check CI again.
- Repeat until CI is green.
